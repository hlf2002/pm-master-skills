# PM 原型生成器

将手绘草图转换为高保真交互原型的技能。

## 工作流程

### 第一步：接收草图
用户会上传一张手绘的原型草图（照片或截图）。用 Read 工具查看图片内容。

### 第二步：识别UI元素
分析草图中的所有UI组件，包括：
- 导航栏 / 标签栏
- 按钮（主按钮、次按钮、图标按钮）
- 输入框（文本输入、搜索框、表单输入）
- 列表项（文字列表、图片+文字列表）
- 图片区域
- 卡片/容器
- 底部导航/侧边栏
- 图标
- 文字标签/标题

识别每个元素的位置（顶部/中部/底部）、尺寸、样式特征。

### 第三步：生成原型代码
调用 frontend-design skill 生成高保真HTML原型：

1. **设计风格选择**：
   - 现代简洁风（适合B端产品）
   - 卡片式设计（适合移动端）
   - 数据仪表盘（适合后台产品）
   - 电商风格（适合零售类）

2. **代码要求**：
   - 单HTML文件，包含内联CSS和JS
   - 使用现代CSS（Flexbox/Grid）
   - 添加基础交互（按钮点击、输入反馈、页面切换）
   - 响应式适配移动端

### 第四步：输出结果
生成后：
1. 在浏览器中打开预览
2. 告诉用户如何预览和分享

## 交互示例

常见可添加的交互：
- 按钮hover/点击效果
- 输入框focus状态
- 列表项点击（可跳转模拟）
- 标签页切换
- 模态框弹出
- 下拉菜单
- 加载状态模拟

## 模板参考

详见 `templates/` 目录下的模板文件。

## 注意事项

- 草图不需要很精美，潦草的线条也可以识别
- 如果草图中有文字标注，优先使用标注内容
- 如果不确定某个元素是什么用途，生成后让用户确认并调整
- 确保生成的代码可以直接在浏览器打开使用
